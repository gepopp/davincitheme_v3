Starter Theme

Als Grundlage für die Implementierung des Themes dient das Gin Tonik Starter Theme von tonik LABS.

Starter Theme Documentation

CSS Framework

Als Grundlage für das Theme Stylesheet wurde das Utility basierte Framework, tailwindcss verwendet. CSS und JS wurde mit dem webpack preprocessor compiliert, dabei wurde das Framework mittels des purgecss Plugins um nicht verwendete Klassen bereinigt, es steht also in der Produktiv-Version nicht das gesamte Framework zur verfügung.

tailwindcss Documentation
Farberweiterungen

Wir haben das Framework um die 4 Hauptfarben des Themes erweitert.

  • golden
  • darkblue
  • lightblue
  • turquise
Damit stehen bei der Inhaltsgestaltung folgende Helferklassen zur verfügung:
  1. Text:text-golden, text-darkblue, text-lightblue, text-turquise
  2. Hintergrund:hg-golden, hg-darkblue, hg-lightblue, hg-turquise
  3. Rahmen:border-golden, border-darkblue, border-lightblue, border-turquise

JS Framework

Zur Implementierung der Frontend-Interaktivität wurde das Vue2.JS Framework verwendet. Für Ajax Calls wurde die axios Bibliothek verwendet.

vue.js Documentation axios Documentation

Kein JQuery:

Achtung: Für nicht eingeloggte User wird kein JQuery geladen, dies dient zur Verbersserung der Ladezeit. Dadurch kann es sein das Plugins nicht wie erwartet funktionieren.

Erforderliche Plugins

Zur Verwaltung der zusätzlichen Metadatenfelder in allen Inhalttypen und Einstellungen verwendet das Theme, das Advanced Custom Fields Plugin in der Pro-Version. Dieses Plugin muss vor dem Theme installiert werden. Das Theme wird mit den Feld-Definitionen als JSON Dateien ausgeliefert. In der ACF Pro version können diese nach der Installation synchronisiert werden.

Theme Einstellungen

Theme Grundeinstellungen

Das Theme verfügt über drei Grundeinstellungen:

  1. Inhlatshintergrund:

    Das hier hochgeladene Bild wird als Hintergrund auf allen Seiten in der Sektion zwischen dem Kopf- und Fussteil, unten eingefügt.

    Sicherheitswarnung

    Zur besseren Ladezeitenoptimierung haben wir mit dem Theme den Upload von .svg Dateien frei geschalten. Mit diesen Dateien kann Schadcode in die Seite eingeschläust werden. Bitte nur vertrauenswürdige Dateien hoch laden.

  2. Abstand Inhlat - Fusszeile

    Diese Einstellung definiert den Abstand des letzten Inhaltes in der Sektion zwischen Kopf- und Fusszeile, zur Fusszeile und damit wie weit der Inhalt das Hintergrundbild überlappt.

  3. Empfänger Kontaktformular

    Die E-Mail Adresse an die, die Daten eines ausgefüllten Kontaktformulars versendet werden.

Kopfzeile

Die Kopfzeile des Themes verfügt über fünf Einstellungen und einen vordefinierten Menüplatz.

  1. Logo:

    Das Logo das in der Desktop-Version in der Banderole ausgegeben wird. Optimale Auflösung 150x150 px oder .svg Datei

  2. Logo Mobil:

    Das Logo das in der Mobilen-Version in der Banderole ausgegeben wird. Optimale Auflösung 150x100 px oder .svg Datei

  3. Hintergrundbild der Kopfzeile

    Das Bild das hinter der Banderole und dem oberen Teil der Seite angezeigt wird, da dieses Bild bildschirmfüllend gestreckt wird sollte es in einer Mindebreite von 2700px hoch geladen wrden und dabei eine Dateigröße on 50 Kb nicht überschreiten.

  4. Kopfbutton Link und Text

    Der Link kann sowohl intern wie auch extern gesetzt werden, Bleibt Link oder Text leer, wird kein Button angezeigt.

  5. Das Hauptmenü

    Das Theme verfügt über einen vordefinierten Menüplatz "Hauptmenü".

Immobilien

Einzelene Immobilien werden über die OpenImmoAPI aus Team3 Importiert.

Immobilien können, mit Projekten über die Felder OpenImmo->GruppenID bei Projekten und TechnischeVerwaltung->Gruppenkennung bei Immobilien, mmiteinander verknüpft werden.

Immobilien Stati
  1. FREI:

    Ist der default Status einer Immobilien

  2. Reserviert:

    Obejekte sind dann reserviert wenn das aktuelle Datum des Seitenaufrufs nicht zwischen den Dati in Objektverwaltung->Ab Datum und Objektverwaltung->Bis Datum liegt. Wenn eines der Datumsfelder leer ist wird dies für die Determinierung durch ein Datum, undendlich in der Vergangenheit bzw. Zukunft ersetzt.

  3. Verkauft:

    Ist bei einer Immobilien das Feld Objektverwaltung->Verfügbar ab mit dem Wort "verkauft" befüllt so wird der Status Verkauft ausgegeben, wobei der Status verkauft den Status reserviert überschreibt.

Frontpage Inhlate

Die hier angelegten Inhlate werden zusammen mit den Immobilien Projekten auf den Seiten mit dem Projekt-Archiv Template ausgegeben.

Team Miglieder

Diese Inhlate werden als automatisch generierte Übersicht ( Archiv ) ausgegeben.

Keine Einzelseiten:

Achtung: Aktuell unterstützt das Template nicht die Ausgabe von Team-Miglieder Einzelseiten.

So funktioniert der Automatismus der Übersichtssteite der Team-Mitglieder
  1. Abfragen der "Team Tags"

    Es werden alle Team Tags abgefragt, Tags bei denen kein Team Member gesetzt wurde werden dabei ausgelassen. Die Tags werden dann nach der eingegebenen Reinfolgennummer, absteigend sortiert.

    Für jedes Tag wird dann im Frontend eine neue Zeile ausgegeben. Diese Zeile enthält in der linken Spalte der Tag Name als Überschirft und der Subtitel ausgegeben.

  2. Abfragen des Team Mitlgliedes

    In Frontend werden dann zu jedem Tag die entsprechenden Team Mitglieder, sortiert nach Datum ausgegeben.

    Bitte beachten sie das die Bilder der Team Mitglieder im Verhältnis 4:3, aus dem Mittelpunkt heraus beschnitten ausgegeben werden.

Seiten Templates

Das Template verfügt, neben den WordPress Standard Template über zwei Sonder-Templates. Templates können in der Seitenleiste des Backends ausgewählt werden.

Indent Template

Beim Indent Template wird der Seiteninhalt nach H2 Überschriften aufgeteilt und in einem zwie Spalten Layout ausgegeben. Bitte beachten sie das leere H2 Überschriften auch als leere Sektionen ausgegeben werden.

Projekt Archiv Template

Im diesem Template werden alle Inhlate der Typen Immobilien Projekt und Frontpage Inhalt ausgegeben. Diese Inhlate werden nach Kategorie gefiltert, diese Filter kann auf der jeweiligen Seite im Backend gesetzt werden und muss bei den Inhalten getagged sein.

Des weiteren kann im Seitenbackend ein Prefix für Tags gesetzt werden, diese Tags, angewendet auf das jeweilige Projekt bestimmen, wie die Inhalte dargestellt werden. Es stehen folgende Tags zur Darstellungssteuerung zur Verfügung:

Box Layouts:
  1. [prefix]-full

    Ganzzeiliges Layout

    • Zusatz: [prefix]-reverese

      Gibt das ganzzeilige Layout mit dem Bild rechts aus.

  2. [prefix]-half

    Halbzeiliges Layout

    • Zusatz: horizontal

      Gibt das halbzeiliges Layout mit dem Bild oben aus, wenn der Tag "horizontal" zusätzlich zu [prefix]-half gesetzt wird.

  3. [prefix]-quarter

    Viertelzeiliges Layout

  4. [prefix]-eighth

    Achtel Layout

    Achtung: Vom eighth Layouts müssen immer 2 aufeinander folgen.

Layout Helfer:

Farbhelfer

  1. [prefix]-darkblue - default
  2. [prefix]-lightblue
  3. [prefix]-turquise

Responsive Helfer

  1. [prefix]-hiddenmobile

    Deises Element wird bis zu einer Bildschirmbreite von 1024px nicht angezeigt.

  2. [prefix]-hiddendesktop

    Deises Element wird ab einer Bildschirmbreite von 1024px nicht angezeigt.

Sortierung

Per default werden die Projekte und Inhalte nach dem Veröffentlichungsdatum sortiert.

Mit ausnahme der Kategorie "Startseite" hier werden die Inhlatytypen nach dem "Reihenfolge" Feld sortiert.
